## Architecture Notes

**Service Pattern Compliance:**
- Follows CMS-Core service pattern (name, init, register exports)
- Registered in boot.js with services.register()
- Available via `context.services.get('ai-registry')`
- CLI commands follow established pattern

**Integration Points:**
- Integrates with core/discovery.js (uses discovered modules)
- Integrates with core/boot.js (initialization sequence)
- Integrates with core/cli.js (command registration)
- Integrates with core/services.js (service registry)

**Design Decisions:**
- Separate from core discovery (AI-specific logic)
- In-memory storage (fast lookups, boot-time discovery)
- Type-based indexing (fast queries by AI type)
- Extensible metadata (capabilities as freeform object)
- Module self-identification (via manifest.json)

**Performance:**
- Map-based structures for O(1) lookups
- Index by type for fast filtered queries
- Minimal overhead (only scans modules with ai: true)
