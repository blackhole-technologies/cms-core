# CMS-Core Contrib Parity Specification

## Project Overview

**Name:** CMS-Core Contrib Parity
**Type:** Zero-dependency Node.js CMS extensions
**Version Target:** 1.1.0
**Existing Codebase:** /Users/Alchemy/Projects/experiments/cms-core (v0.0.80+)

This specification defines features to add Drupal contrib module functionality to CMS-Core, based on analysis of 67 Drupal-CMS contrib modules.

## Technology Stack

- **Runtime:** Node.js 20+ (ES modules)
- **Dependencies:** ZERO npm packages (all built from node: built-ins)
- **Storage:** Flat-file JSON
- **Server:** node:http
- **Architecture:** Service pattern (init/register exports)

## Core Constraint

**CRITICAL:** This project has NO npm dependencies by design. All functionality must be built using only Node.js built-in modules.

## Feature Count

**feature_count: 200**

This is a LARGE complexity project adding contrib module parity.

## Implementation Notes

### AI Integration Pattern
```javascript
// Provider registration
export function init(ctx) {
  const ai = ctx.services.get('ai');
  ai.registerProvider('openai', {
    models: ['gpt-4', 'gpt-3.5-turbo'],
    operations: ['chat', 'embeddings'],
    async chat(messages, model, options) { ... }
  });
}
```

### ECA Rule Pattern
```javascript
// Rule stored as JSON config
{
  "id": "notify-on-publish",
  "events": ["content:create"],
  "conditions": [
    { "plugin": "content_type_is", "value": "article" },
    { "plugin": "field_value_is", "field": "status", "value": true }
  ],
  "actions": [
    { "plugin": "send_email", "to": "[author:email]", "subject": "Published!" }
  ]
}
```

### Function Call Pattern
```javascript
// AI function call plugin
export const functionCall = {
  name: 'create_content',
  description: 'Create new content in the CMS',
  parameters: {
    type: { type: 'string', required: true },
    data: { type: 'object', required: true }
  },
  async execute({ type, data }, ctx) {
    const content = ctx.services.get('content');
    return content.create(type, data);
  }
};
```
