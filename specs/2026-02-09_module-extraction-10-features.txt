# CMS-Core: Module Extraction
# Move feature code out of core/ into proper modules that register via hooks.
# CLAUDE.md says "Don't put feature code in core/" but 60+ files still do.

## Architecture Context
core/ contains ~91 files. Many are genuine core services (boot.js, server.js, router.js,
hooks.js, services.js, database.js, cache.js, auth.js, etc.). But 60+ files are feature
code that belongs in modules/ — things like views.js, taxonomy.js, workspaces.js,
layout-builder.js, search.js, feeds.js, comments.js, etc.

This spec extracts the most impactful feature files into proper modules. Each extracted
module registers its routes, hooks, and services through the standard module pattern
instead of being hardcoded into the boot sequence.

The boot bridge (separate spec) must be in place for this to work — modules need to
register via hooks and the service container.

## Read These Files First
- core/boot.js — understand how core files are currently loaded
- modules/hello/index.js — example of a proper module (hook registration pattern)
- modules/admin/index.js — example of a complex module
- CLAUDE.md — architecture rules including "Don't put feature code in core/"

## Important Constraints
- Do NOT delete core/ files — move them to modules/ and create a shim in core/ that imports from the module for backward compatibility
- Each module must have: index.js (hook_boot registration), and the moved logic
- Modules register routes via hook_routes, services via hook_services
- Test that the CMS still boots and responds after each extraction

## Technology Stack
- Runtime: Node.js (zero npm dependencies)
- Port: 3001

feature_count: 10

## Phase 1: High-Value Extractions (Features 1-4)

1. Extract core/views.js into modules/views/. Create modules/views/index.js with hook_boot that registers the views service and routes. Move the views logic to modules/views/views.js. Replace core/views.js with a thin shim that imports from modules/views/ and re-exports for backward compatibility. Add a console.warn deprecation notice in the shim. Enable 'views' in config/modules.json.
