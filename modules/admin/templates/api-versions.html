Version Negotiation

Clients can specify the desired API version using these methods (in priority order):

  1. URL Prefix /api/v2/content/...
  2. X-API-Version Header X-API-Version: 2
  3. Accept Header Accept: application/vnd.cms.v2+json
  4. Query Parameter ?api_version=2
  5. Default Falls back to {{defaultVersion}}

Response Headers

Header Description Example
X-API-Version The API version used for this response v2
Deprecation Present when using a deprecated version true
Sunset Date when deprecated version will be removed Sat, 01 Jun 2024 00:00:00 GMT
Link Points to successor version when deprecated </api/v2/>; rel="successor-version"
