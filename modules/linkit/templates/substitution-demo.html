🔗 Link Substitution Service

Store entity references, not raw URLs

What is Link Substitution?

Instead of storing raw URLs in content, we store entity references as tokens. When rendering, tokens are replaced with current entity URLs.

Why use tokens?
  • ✅ Links stay valid if entity URLs change
  • ✅ Path alias updates don't break links
  • ✅ URL structure changes are handled automatically
  • ✅ Graceful handling of deleted entities

Token Format

Tokens use the format: {entity:type:id}

Examples:

{entity:content:abc123} → Link to content with ID abc123
{entity:user:xyz789} → Link to user with ID xyz789
{entity:media:file456} → Link to media with ID file456
