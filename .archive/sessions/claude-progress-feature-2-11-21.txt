## Technical Notes

**Database Schema Pattern**:
```json
{
  "_schema": {
    "fields": { "id": { "type": "serial", ... } },
    "primary_key": ["id"],
    "indexes": { "name": ["name"] }
  },
  "_rows": [ {...}, {...} ]
}
```

**Plugin Discovery Pattern**:
- Module structure: `modules/{module}/plugins/{type}/{Plugin}.js`
- Definition export: `export const definition = { id, label, ... }`
- Factory export: `export default function create(config, id, def, services) { ... }`
- Metadata attached: `_module`, `_path`, `_factory`

**Hook Alter Pattern**:
- Register: `hooks.onAlter('hook_name', handler)` (no _alter suffix)
- Invoke: `hooks.alter('hook_name', data, context)`
- Chain: Each handler receives result of previous, can return modified data
- Mutation: Returning undefined preserves data (allows in-place mutation)
