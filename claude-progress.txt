1. **core/boot.js**: Added bridge between config/content-types.json and content system.

   causing CLI/API commands to reject them. Now registers during boot after

   `created` and `updated` are required+readonly, auto-set by the system.
   Without this fix, content:create fails requiring user to provide system fields.

### Changes Made
1. **core/constraints.js**: New Drupal-inspired constraint plugin architecture.
   Plugin registry with register/get/has/list. Violation objects with
   constraint, field, message, value, code. validate() collects all violations.
   validateOrThrow() for content save integration. CLI commands for listing
   and checking constraints.

2. **core/boot.js**: Import and initialize constraints service after validation.
   Register as service, inject into content via setConstraints().

3. **core/content.js**: Added constraintService injection via setConstraints().
   Modified create() and update() to use constraint validation when available,
   falling back to legacy validateData() if not. Added getType() export.
