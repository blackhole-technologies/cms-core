## Workflow and Publishing

Workflows control how content moves from draft to published state with optional approval steps.

### Content States

Content progresses through different states:

1. **Draft** - Initial creation
2. **Review** - Submitted for approval
3. **Published** - Live on the site
4. **Archived** - Removed but retained

### Basic Workflow

The default workflow is simple:
- Create content as **Draft**
- Edit and refine
- **Publish** when ready

### Advanced Workflow

For teams, enable approval workflows:

1. Author creates content (Draft)
2. Author submits for review
3. Editor reviews and approves/rejects
4. If approved, content is published

### Workflow Permissions

Control who can perform workflow actions:
- **Create Draft** - All content authors
- **Submit for Review** - Authors
- **Approve** - Editors only
- **Publish** - Editors and Admins
- **Archive** - Admins only

### Scheduled Publishing

Schedule content to publish automatically:
1. Set a **Publish Date** in the future
2. Save the content
3. Content will auto-publish at the specified time

### Revisions and History

Track all changes to content:
- Each save creates a new revision
- View revision history
- Compare revisions
- Revert to previous versions

### Notifications

Configure workflow notifications:
- Notify editors when content needs review
- Notify authors when content is approved/rejected
- Send reminders for pending approvals

### Tips

- Use workflow for content that needs oversight
- Schedule posts to publish during peak hours
- Review revision history to track changes
- Set up email notifications for your team
